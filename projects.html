Cosmic: A Journey Among Shadows

This is a commercial project created and published by the indie game studio King's Pleasure. I formed part of the development team for over a year, with focus on C# scripting. My contribution includes implementation of some of the main character skills, improvement of the gamepad input, parts of the UI as well as parts of the translation and saving systems. I was also writing custom actions for visual scripting and worked with Steam API.

During the project I had an opportunity to use programming patterns such as: singleton pattern, observer pattern and object pooling. I was also doing my best to apply whenever I could the SOLID principles. Moreover, I worked with many Unity plug-ins, including Playmaker, DOTween, Odin Inspector or I2 Localization. We used Plastic SCM for source control.

Watch the trailer on YouTube.
You can purchase the game on the official Steam page.

As Astra

This is my personal project with focus on 2D platforming, tilemaps and environmental challenges. The game consists of a short story and 3 unique levels of my own design. I developed and implemented gravity-related mechanic that strips the player of the core abilities and forces them to use the level itself to their advantage. I also included puzzle-like mechanics involving movable boxes and transporting platforms. The last part of the game features a final boss battle and special scripted events.

In the project I followed the OOP principles, used inheritance and polymorphism, as well as interfaces and object pooling.

You can play the game online on Itch.io.
The whole project is available on my GitHub.

Math or Words?

This is my personal project developed in the relatively new .NET framework for multiplatform mobile and desktop apps - MAUI. The game has 2 modes: math-oriented and words-oriented. The first one tests player's proficiency in different math operations. The second one consists of word formation and working with associations. Player scores are stored in a local database and can be browsed on demand.

The project was as an opportunity to learn about consuming APIs, in this case the Wikipedia API that serves as a validator of player answers. I used asynchronous programming when retrieving data from the Wikipedia and applied all CRUD operations on the database. I also took advantage of dependency injection for the database connection.

You can download the desktop build of the app here.
The whole project is available on my GitHub.
